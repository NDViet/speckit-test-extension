# Generate QA Test Plan

Produce a complete QA test plan document at `specs/NNN-*/testplan-NNN-feature.md`.
This is the test plan the engineer links in the PR description before merge.

The plan is generated from:
- `spec.md` — AC-N entries, actor, out-of-scope, [NEEDS CLARIFICATION]
- `plan.md` — technical environment, integration points, API contracts
- `tasks.md` — T00x [P] test tasks and automation tags
- `constitution.md` — test framework, QA non-negotiables, coverage thresholds

## User Input

```text
$ARGUMENTS
```

Consider user input before proceeding. The user may specify:
- Target audience (e.g., "QA team", "developer", "PM sign-off")
- Scope (e.g., "regression only", "smoke test", "full plan")
- Output filename override (e.g., `testplan-001-hotel-filter.md`)
- Specific ACs to include or exclude

## Prerequisites

1. Confirm you are inside a git repository.
2. Locate `specs/NNN-*/spec.md`. Ask if ambiguous.
3. Read `spec.md` fully — actor, all AC-N, out-of-scope, [NEEDS CLARIFICATION].
4. Read `plan.md` — architecture, integration points, external services, API error codes.
5. Read `tasks.md` — all T00x [P] test tasks and their automation tags.
6. Read `.specify/memory/constitution.md` — test framework, QA non-negotiables.
7. Check whether `testplan-NNN-*.md` already exists — if so, update it rather than overwriting.

## Rules

- **AC-driven** — every test case in the plan must reference an AC-N; no freestanding test cases
- **Traceability matrix is mandatory** — the AC traceability matrix (Section 4) must appear in every plan; it is the document's primary QA artefact
- **No tech leakage in test cases** — test case descriptions describe observable behaviour, not implementation (same rule as spec.md ACs)
- **Out-of-scope excluded** — do not generate test cases for anything listed under `## Out of scope` in spec.md
- **P1 threshold explicit** — always state "All P1 test cases must pass" as the acceptance threshold for every AC section
- **Framework from constitution** — always reference the actual team test framework from constitution.md, not generic placeholders
- **Exit criteria are non-negotiable** — always include the five exit criteria: P1 tests pass, zero P1 defects, /speckit.test.coverage ≥ threshold, /speckit.test.gaps = 0 critical, /speckit.analyze clean
- **Update, don't overwrite** — if testplan-NNN-*.md already exists, update it to reflect spec changes rather than replacing it wholesale
- **[NEEDS CLARIFICATION] items are risks** — any unresolved [NEEDS CLARIFICATION] from spec.md must appear in the Risk Assessment with "Escalate to PM" mitigation
