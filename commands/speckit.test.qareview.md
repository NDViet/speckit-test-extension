# Consistency + Quality Verdict

Runs on `after_analyze` (advisory pre-impl read) and `after_implement` (formal pre-merge gate, with real test results).

**Writes:** `FEATURE_DIR/review.md` only (overwritten each run). All other artefacts read-only.
**Source of truth for consistency:** `/speckit-analyze`'s report (lifted from conversation; not re-derived).

## User Input

```text
$ARGUMENTS
```

Flags: `--strict` (Majors → Blockers), `--story US1`, `--no-write` (chat-only; tracking is the default).

## Prerequisites

1. `.specify/scripts/powershell/check-prerequisites.ps1 -Json -RequireTasks -IncludeTasks` → parse `FEATURE_DIR`.
2. Read `spec.md`, `plan.md` (incl. `## Testing Strategy`), `tasks.md`, `test-plan.md`, `checklists/test.md`, `.specify/memory/constitution.md`. Scan `tests/`.
3. **Lift `/speckit-analyze`'s in-conversation report** when available: finding table (A1, D1…), Coverage Summary, Unmapped Tasks, Constitution Alignment.

## Checks

### 1. Requirement-level coverage

For every gated item (P1 `US{n}-AS{m}`, `FR-###`, buildable `SC-###`), rate the mapping:

| Rating | Criteria |
|---|---|
| **Strong** | ≥1 non-stub test names the item ID + exercises documented behaviour |
| **Medium** | Test names the item; partial coverage (happy path only) |
| **Weak** | Test in the right area but no item ID — traceability broken |
| **Stub** | Only the qaprep failing stub exists — 0% coverage |
| **Missing** | No test file |

Constitution threshold decides which ratings escalate to Blocker (default: every P1 → Strong; SC perf may be Medium).

### 2. Stub scan
Unchanged `qaprep` failing stub + P1 item → **Blocker**.
Body matches "stub by description" task → **Major**.

### 3. Cross-reference `/speckit-analyze`
- CRITICAL → Blocker (prefix `[ANALYZE]`, preserve analyze ID).
- Constitution-aligned violation → Blocker.
- Requirement with no mapped task (per analyze) → Blocker.
- Unmapped task (per analyze) → Major (scope drift).

### 4. Traceability chain
Verify end-to-end: `spec item → plan.md Testing Strategy row → tasks.md test task → test file with ID in header → test-plan.md matrix row`. Broken link → Major.

### 5. Artefact presence
- `test-plan.md` exists + every gated item in matrix → else Blocker.
- `checklists/test.md` exists + gated rows ticked → else Major (Blocker under `--strict`).
- Constitution-required test-plan sections present → else Major.

## Rules

- **Single write target: `review.md`** — overwritten each run; chat body and file body identical (file adds metadata header). `--no-write` skips file (rare).
- **Overwrite, not append** — history lives in git; file is the *latest* state.
- **Lift, don't re-derive, consistency findings** — `/speckit-analyze` is the source of truth.
- **`after_implement` is the formal gate** — it has real test results; `after_analyze` runs advisory.
- **Stubs ≠ coverage** — unchanged qaprep stub = `Stub = 0%`.
- **Traceability needs the item ID *in the test*** — without it, max rating is Weak.
- **Constitution MUSTs always escalate to Blocker** — cannot be silenced.
- **Missing test-plan.md / checklist / scaffolds → first Blocker is "re-run after-tasks (qaprep)".**
- **Explicit verdict** — always `Gate: PASS|BLOCKED` + `SPECTEST QAREVIEW:` one-liner.
