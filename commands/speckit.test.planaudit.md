# Plan-time Unit Test Strategy ‚Äî `after_plan` hook

Decides the unit/contract test layer at planning time so `/speckit-tasks` materializes the cases as real tasks and `tasksaudit` runs as a thin verification pass.

**Scope:** unit + contract only. Integration / E2E / regression / perf / a11y belong to `qaprep`'s `test-plan.md`.
**Writes:** `plan.md` ¬ß `## Testing Strategy` only. Never touches `spec.md`, `tasks.md`, tests, or other plan.md sections.
**Hook default:** write. `--dry-run` previews.

## User Input

```text
$ARGUMENTS
```

Flags: `--dry-run` (preview), `--story US1` (scope to one story), `--advisory` (banner only, TDD waived).

## Prerequisites

1. `.specify/scripts/powershell/check-prerequisites.ps1 -Json -RequireSpec -IncludePlan` ‚Üí parse `FEATURE_DIR`.
2. Read `spec.md` (Acceptance Scenarios, FR, Success Criteria, Edge Cases), `plan.md` (stack, layout, declared test framework), `.specify/memory/constitution.md` (TDD policy).
3. Do **not** read `tasks.md` ‚Äî it does not exist yet.

## Outline

### 1. Policy
- **Default:** TDD mandatory ‚Äî every P1 AS and FR needs ‚â•1 unit/contract case.
- **Constitution** may escalate layers (record principle name).
- `--advisory` records "TDD waived" banner, no gating change.

State the active policy at the top of the rendered section.

### 2. Testable items (from spec.md)

| Source | ID | Gated here |
|---|---|---|
| `### User Story N` ‚Üí Acceptance Scenarios | `US{N}-AS{M}` | **Yes** (P1). P2/P3 advisory. |
| `### Functional Requirements` | `FR-###` | **Yes** |
| `## Success Criteria` (perf/security/availability only) | `SC-###` | Advisory ‚Äî deferred to `qaprep` |
| `### Edge Cases` | ‚Äî | Case candidates under their FR/AS |

### 3. Framework & paths
Infer from plan.md (language ‚Üí conventional framework; declared paths). If undeclared, propose one and tag `NEEDS CONFIRMATION`.

### 4. Generate cases
For each in-scope item, emit ‚â•1 happy-path case and ‚â•1 error/edge case (when an edge applies). Each case has: Item ID, Layer (`unit` default; `contract` for API/schema), Case name, Arrange/Act/Assert (map G/W/T directly when AS), concrete test path, mocking boundary.

Append at end of plan.md (or replace existing block between `## Testing Strategy` and the next `## ` heading ‚Äî never duplicate).

### 6. Report


`--dry-run`: `SPECTEST PLAN (dry-run): would write 10 cases; plan.md unchanged.`
Missing framework: append `‚Äî framework proposed as Vitest, NEEDS CONFIRMATION`.

## Rules

- **Idempotent** ‚Äî re-running replaces the block in place; never duplicates.
- **Modify only plan.md / Testing Strategy section.**
- **Unit + contract only** ‚Äî never plan higher layers as gated here.
- **No invented items** ‚Äî every case traces to a real `US{N}-AS{M}` or `FR-###`.
- **Concrete paths** ‚Äî no `tests/somewhere/*`.
- **Constitution may escalate, never silently waive** ‚Äî waiving requires explicit `--advisory`.
